📁 MusicPlayer Project Structure Explained
🟢 Root Level Files
App.tsx
🚀 Entry point of the app

Loads navigation

Wraps providers (store/navigation/theme)

Starts the whole UI

👉 Think: Main() of React Native

app.json
Expo configuration file

App name, icon, splash screen, permissions

package.json
Lists dependencies

Scripts like:

npm start
npm run android
npm run ios
babel.config.js
Babel setup for React Native/Expo

Needed for compiling JS/TS

README.md
Documentation of the project

Setup steps + features

📂 Native Folders
android/
Native Android project generated by Expo

Required for building APK

ios/
Native iOS project files

👉 Usually do not edit manually

📂 api/
jioSaavnApi.ts
Handles API calls to JioSaavn

Contains:

searchSongs()
getSongDetails()
getArtistSongs()
👉 Separates network logic from UI

📂 components/
Reusable UI building blocks.

AlbumBackground.tsx
Shows blurred album art behind player

Used in PlayerScreen

EmptyState.tsx
Displayed when:

no songs
no results
empty queue
MiniPlayer.tsx
Persistent bottom music bar

Shows:

song title
play/pause
album image
Syncs with full PlayerScreen

⭐ Very important component

PlayerControls.tsx
Contains music buttons:

previous
play/pause
next
shuffle
repeat
Section.tsx
Reusable section wrapper for Home screen
Example:

Trending
Recently Played
Recommended
SeekBar.tsx
Music progress slider

Handles:

current time
duration
seek position
SongCard.tsx
Displays a single song item
Used in:

Home list
Search results
Queue
📂 hooks/
Custom React hooks.

Typical use:

usePlayer()
useQueue()
useAudio()
👉 Keeps logic reusable and clean.

📂 screens/
Each file = one full app page.

HomeScreen.tsx
Landing page

Shows:

song list
sections
pagination
PlayerScreen.tsx
Full music player UI

Shows:

album art
seekbar
controls
background
⭐ Core feature of the app

QueueScreen.tsx
Shows upcoming songs

Allows:

reorder
remove
view current track
SearchScreen.tsx
Search songs/artists/albums

Uses API search endpoint

📂 services/
Contains helper logic like:

audio playback handler
download manager
format time utilities
👉 Business logic separate from UI.

📂 store/
Global state management (Zustand).

Stores:

currentSong
isPlaying
queue
progress
repeatMode
shuffle
👉 Keeps PlayerScreen + MiniPlayer synced.

⭐ This is critical for assignment requirement

📂 theme/
colors.ts
Central color palette
Example:

background
card
accent
text
subtext
👉 Makes UI consistent across app.

🧠 How Everything Works Together
Flow Example:
1️⃣ User taps song in SongCard
→ updates state in store

2️⃣ Store updates:

currentSong = newSong
isPlaying = true
3️⃣ Both:

MiniPlayer
PlayerScreen
automatically re-render

4️⃣ Audio handled via services

5️⃣ UI styled using theme/colors.ts
